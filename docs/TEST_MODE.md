# TEST MODE

BirgeFund работает только в демонстрационном режиме.

Платформа не является:

- инвестиционной системой;
- платёжной организацией;
- электронным кошельком;
- банком;
- финансовым продуктом.

## Что запрещено в demo-версии

В demo-версии запрещено:

- принимать реальные деньги;
- вводить реальные банковские карты;
- запрашивать CVV;
- запрашивать номер банковской карты;
- выполнять выплаты авторам;
- показывать настоящий баланс пользователя;
- обещать доход;
- обещать проценты;
- обещать дивиденды;
- показывать инвестиционную модель;
- создавать ощущение настоящей оплаты.

## Что разрешено

Разрешено:

- поддержать проект в тестовом режиме;
- выбрать mock-метод оплаты;
- создать `payment_attempt`;
- создать `ledger_entries`;
- показать тестовый чек;
- показать историю тестовых операций;
- сделать test refund;
- записать действия в audit log.

## Mock payment modal

В модальном окне можно показывать:

~~~text
Сумма
Метод оплаты: TEST_CARD / TEST_WALLET / TEST_BANK
Кнопка: Оплатить в тестовом режиме
Предупреждение TEST MODE
~~~

Нельзя показывать:

~~~text
Номер карты
CVV
Срок карты
Имя владельца карты
~~~

## Обязательный текст

На страницах, связанных с поддержкой проекта, должен быть текст:

~~~text
Поддержка проекта не является инвестицией.
Доход не гарантируется.
Платформа работает в тестовом режиме.
Реальные деньги не списываются.
~~~

## Idempotency

Каждая mock payment операция должна иметь `idempotency_key`.

Если пользователь нажал кнопку оплаты два раза:

- новая операция не создаётся;
- возвращается уже существующий `payment_attempt`;
- ledger не дублируется.

## Refund

Refund не удаляет старую операцию.

Пример:

~~~text
PROJECT_GROSS +10000
PLATFORM_FEE -700
PROJECT_NET +9300
REFUND -10000
~~~

Refund может делать только пользователь с нужными правами:

- Super Admin;
- Financial Manager.

Каждый refund требует причину.

Предзаказать товар
Получить отчёт
Поддержать социальную инициативу

## Что переносится во второй этап

Во второй этап переносится:

- реальные платежи;
- интеграция с банком;
- вывод денег авторам;
- investment/profit-sharing;
- реальный электронный кошелёк;
- сложная KYC-проверка;
- автоматическая проверка документов.
